# 配置恢复与标签删除复盘

## 问题

`config set db.path` 写入不可用路径后，所有命令都会先初始化 SQLite，导致 `config reset` 无法恢复配置。

标签配置读取使用 `raw.get("tags") or DEFAULT_TAGS`，当标签列表被删到空数组时，下次读取会恢复默认标签，造成删除命令表现为成功但状态被隐式回滚。

## 根因

- CLI 运行上下文在命令分发前无条件初始化数据库，没有区分纯配置命令和数据命令。
- 配置读取把“字段不存在”和“字段存在但为空”混为一谈。

## 修复

- 为 `create_app_context` 增加 `initialize_database` 参数，`config get/set/reset` 不初始化数据库。
- 捕获 `OSError`，让文件系统路径错误以 CLI 错误退出而不是抛出异常。
- 标签读取只在 `tags` 字段不存在时使用默认标签。
- 禁止保存空标签列表，删除最后一个标签时返回明确错误。

## 预防规则

- 纯配置恢复命令不得依赖被恢复对象处于可用状态。
- 配置读取必须区分缺失值、空值和显式配置值。
- 对配置恢复和边界删除行为补充回归测试。
